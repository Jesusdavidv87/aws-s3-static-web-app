# AWS S3 Static Web App

## Overview
This project demonstrates how to host a static web application using Amazon S3.

I built and deployed a simple cloud-themed web app and made it publicly accessible through Amazon S3. The goal of this project was to strengthen my understanding of static website hosting, object storage, permissions, and cost-efficient cloud architecture.

## What I built
- Created an Amazon S3 bucket
- Uploaded static website files
- Configured object permissions for public access
- Deployed a live website accessible through an S3 object URL
- Customized the page to reflect my cloud learning journey

## Why this matters
This project shows how simple applications can be deployed in the cloud without managing servers.

Using S3 for static hosting provides:
- Lower cost than running traditional servers
- High scalability for static content
- Reduced operational overhead
- Fast deployment for lightweight web applications

## AWS concepts reinforced
This project helped reinforce core AWS Solutions Architect Associate concepts, including:
- Amazon S3 object storage
- Static website hosting
- Public access and permissions
- Cost-efficient architecture decisions
- Choosing the right AWS service for the workload

## Architecture
- **Frontend:** HTML, CSS, JavaScript
- **Cloud Service:** Amazon S3
- **Hosting Model:** Static website hosting

## Key takeaway
One of the biggest lessons from this project is that not every application needs compute resources like EC2. For static workloads, S3 can be a simpler, more scalable, and more cost-effective solution.
